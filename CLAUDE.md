# Lorecraft

AI co-author for tabletop RPG Game Masters. Reads an Obsidian vault and
generates lore-consistent campaign content via a CLI (MVP) and later a
locally hosted web UI (v0.4).

## Project layout

- `app/`           — Next.js App Router (web UI, v0.4+). API routes for
                     Vercel AI SDK streaming live in `app/api/`
- `src/vault/`     — everything that touches the vault filesystem: file reads,
                     wikilink resolution, template parsing, frontmatter handling
- `src/agent/`     — dynamic system prompt assembly, context budget manager,
                     generation loop
- `src/llm/`       — LLM provider instantiation, token reporting. Single file
                     that the rest of the codebase imports — never import a
                     provider adapter directly elsewhere
- `src/cli/`       — CLI entry point, slash command parsing, chat REPL
- `src/__tests__/` — integration tests and fixture vault
- `docs/`          — all planning and architecture documents (reference below)

## Stack

- Language: TypeScript strict mode
- Package manager: pnpm
- Framework: Next.js App Router (see AGENTS.md warning — read
  node_modules/next/dist/docs/ before touching app/ code)
- AI SDK: Vercel AI SDK (latest)
- LLM provider: Amazon Bedrock via @ai-sdk/amazon-bedrock — abstracted,
  swap via .env
- Testing: Vitest (unit + integration), Playwright added at v0.4 for E2E
- Vault access: Node.js fs (direct reads, never writes without approval)

## Build & run

```bash
pnpm install
pnpm dev          # Next.js dev server (web UI, v0.4+)
pnpm cli          # CLI: npx ts-node src/cli/index.ts
pnpm build        # Production build
pnpm lint         # ESLint
pnpm typecheck    # tsc --noEmit
pnpm test         # vitest run
pnpm test:watch   # vitest
```

## Core invariants

- Never write to the vault without explicit GM approval
- Read operations autonomous; write operations require a confirmation step
- Context budget manager required before any vault traversal —
  never send unbounded context to the LLM
- Token consumption (input + output + estimated cost) reported after
  every generation
- LLM provider swappable via single .env change — no provider-specific
  types outside src/llm/provider.ts

packages, extract to:
  packages/core/    <- src/vault/ + src/agent/ + src/llm/
  packages/cli/     <- src/cli/
Then add turbo.json. Do not create packages/ prematurely.
